Using the Errai Forge Addon

Downloading Forge and the Forge Addon

If you don't already have JBoss Forge installed on your machine, you'll need to download it to use Errai's addon. Download the latest version of JBoss Forge here.
Once installed, you need to add the Errai Forge Addon to your installation to create a new Errai app. Run the following command in your terminal window: forge --install org.jboss.errai.forge:errai-forge-addon,3.1.1-SNAPSHOT

Using Forge to create an empty Errai App

This section will outline the steps you need to complete to create your own Errai app.

1. Run Forge from your terminal:
forge You should see a prompt that looks like this (where home is the directory in which you started Forge): [home]
2. Run the following command to create a new project:
[home] project-new --named errai-forge-demo --topLevelPackage org.jboss.errai.demo --version 1.0.0-SNAPSHOT --finalName "Errai Forge Demo" --type war --buildSystem Maven In the above parameters:
  • --named is the artifactId.
  • --topLevelPackage is the groupId.
  • --version defaults to 1.0.0-SNAPSHOT.
  • --finalName is the human-readable name of your project.
  • --type is the packaging type of your project.
  • --buildSystem is the build system used by your project.
Forge will now prompt you from within your project: [errai-forge-demo]
3. Run the errai-setup command to install the core dependencies and configurations required for all Errai features:
errai-setup --versionSelect 3.1.1-SNAPSHOT --logicalModuleName org.jboss.errai.demo.ErraiDemo --moduleName ErraiDemo In the above parameters:
  • --versionSelect is the version of Errai you wish to use.
  • --logicalModuleName is the logical name of the GWT module for your project.
  • --moduleName is a short name used for referencing your GWT module in urls.
You can also just type errai-setup instead, and the Errai Forge Addon will prompt you with an interactive interface to enter the rest of the values. TIP: If you are running errai-setup on a pre-existing project with a GWT module you can replace the --logicalModule option with --moduleSelect followed by the logical name of your GWT module. IMPORTANT: Don't forget to unzip a copy of Wildfly 8 into the target/ directory of your new app for debugging purposes.
4. Add Errai Features
The errai-add-features command adds dependencies and configurations for any selected modules in Errai.
To see a list of Errai features, type the following command and press TAB to auto-complete the available Errai modules: errai-add-features --featureSelect Click the links on the left side to learn more about each of the available modules and how to use them.

TIP: Use the errai-remove-features command in the same way to remove a feature that you have previously added to your project. You will not be able to remove a feature if another installed feature depends upon it.

5. Generate Test Classes
The Errai Forge Addon can generate GWT Mockito tests for unit testing and GWTTestCase tests for integration testing.
Unit Testing
To generate unit tests, run the following command on your Errai project in the Forge terminal: errai-add-unit-test --testableClassName org.jboss.errai.MyClass --testClassName org.jboss.errai.unittest.MyIntegrationTest
  • --testableClassName is the fully-qualified name of the Java class being tested.
  • --testClassName is the fully-qualified name of the test class being generated.
Integration Testing

IMPORTANT: Integration tests require Errai CDI. You will not be able to create integration tests until you add Errai CDI to your project using the errai-add-features command.

For integration tests, run the following command: errai-add-integration-test --testClassSimpleName MyIntegrationTest --testPackageName org.jboss.errai.integrationtest
  • --testClassSimpleName is the simple name of the test class being generated.
  • --testPackageName is the root package for the test GWT module.
