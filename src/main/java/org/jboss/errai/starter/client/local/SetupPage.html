Setting up your environment

Java Development Kit

We recommend using version 7 of the Oracle JDK.

Maven

Maven is a dependency management tool that we will use to build our Errai Application. In order to install and use Maven, you will need to have the JDK installed. Errai requires Maven version 3+. Instructions for downloading and installing Maven are on this page.
Run mvn --version on the command line to ensure that you have the correct version installed.

WildFly Application Server

WildFly is an Application Server which we will use to host our Errai Application locally. In the tutorial we will be using WildFly 8.1.0.Final. You can download a zipped copy of WildFly 8 from this link. No installation is necessary. Simply unzip it to wherever you would like the applicationâ€™s directory to be located.

Your First App

There are two ways for you to create your first Errai application. You can use our sample app, Errai Tutorial, as a starting point, or you can create a barebones Errai app using JBoss Forge, that will have all the basic dependencies in place for you. Both options are explained in detail below.

Errai Tutorial

The Errai Tutorial is a basic sample application that showcases all the major features of Errai. You can use it both as an example to explore Errai's functionality and as a base upon which to build your own more complex web app. The Errai Tutorial can be downloaded from the GitHub repository here.

JBoss Forge

To get you started with your first Errai app, we will use JBoss Forge and the Errai Forge Addon to create a new app with the basic dependencies already in place. Download the latest version of JBoss Forge here.

Once installed, you need to add the Errai Forge Addon to your installation to create a new Errai app. Run the following command in your terminal window:
forge --install org.jboss.errai.forge:errai-forge-addon,$VERSION where $VERSION is replaced by the latest Errai version (currently 3.1.1-SNAPSHOT).

1. Run Forge from your terminal:
forge You should see a prompt that looks like this (where home is the directory in which you started Forge): [home]
2. Run the following command to create a new project:
[home] project-new --named errai-forge-demo --topLevelPackage org.jboss.errai.demo --version 3.1.1-SNAPSHOT --finalName "Errai Forge Demo" --type war --buildSystem Maven" In the above parameters:
  • --named is the artifactId.
  • --topLevelPackage is the groupId.
  • --version defaults to 1.0.0-SNAPSHOT.
  • --finalName is the human-readable name of your project.
  • --type is the packaging type of your project.
  • --buildSystem is the build system used by your project.
Forge will now prompt you from within your project: [errai-forge-demo]
3. Run the errai-setup command to install the core dependencies and configurations required for all Errai features:
errai-setup --versionSelect 3.0-SNAPSHOT --logicalModuleName org.jboss.errai.demo.ErraiDemo --moduleName ErraiDemo In the above parameters:
  • --versionSelect is the version of Errai you wish to use.
  • --logicalModuleName is the logical name of the GWT module for your project.
  • --moduleName is a short name used for referencing your GWT module in urls.
You can also just type errai-setup instead, and the Errai Forge Addon will prompt you with an interactive interface to enter the rest of the values. TIP: If you are running errai-setup on a pre-existing project with a GWT module you can replace the --logicalModule option with --moduleSelect followed by the logical name of your GWT module.
Add Errai Features
The errai-add-features command adds dependencies and configurations for any selected modules in Errai.
To see a list of Errai features, type the following command and press TAB to auto-complete the available Errai modules: errai-add-features --featureSelect
